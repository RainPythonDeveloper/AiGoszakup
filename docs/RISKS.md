# Перечень рисков и ограничений

## 1. Риски данных

### 1.1 Качество данных API
- **Риск:** API OWS v3 может возвращать неполные или некорректные данные (NULL поля, несогласованные ссылки между сущностями)
- **Вероятность:** Средняя
- **Влияние:** Снижение точности аналитики
- **Митигация:** data_quality_log таблица фиксирует все проблемы качества; ETL pipeline проверяет ключевые поля; raw_data хранит оригинальные ответы API для повторной обработки

### 1.2 Задержка обновления данных
- **Риск:** Инкрементальное обновление через journal endpoint может отставать от публикации на портале
- **Вероятность:** Низкая
- **Влияние:** Ответы агента могут не учитывать самые свежие закупки (до 24 часов)
- **Митигация:** Airflow DAG запускается каждые 12 часов; в ответе указывается дата последней синхронизации

### 1.3 Неполнота данных за 2026 год
- **Риск:** Данные за текущий год неполные — год ещё не завершён
- **Вероятность:** Гарантирована
- **Влияние:** Тренды и сравнения по годам могут быть искажены
- **Митигация:** Агент указывает в ответе ограничения по периоду; нормализация на количество месяцев

### 1.4 Изменение структуры API
- **Риск:** OWS v3 API может изменить формат ответов или эндпоинты
- **Вероятность:** Низкая
- **Влияние:** Остановка загрузки данных
- **Митигация:** raw_data сохраняет JSON; версионирование схемы; мониторинг ошибок API

## 2. Риски аналитики

### 2.1 Ограничения Fair Price
- **Риск:** Формула Fair Price (Median × RegCoeff × CPI × SeasonCoeff) может не учитывать:
  - Различия в комплектации товара (разные спецификации под одним ENSTRU)
  - Объёмные скидки (крупные партии дешевле)
  - Уникальные условия поставки (срочность, удалённость)
- **Вероятность:** Средняя
- **Влияние:** Ложные срабатывания при выявлении завышенных цен
- **Митигация:** confidence level (high/medium/low); в ответе указывается размер выборки

### 2.2 Сезонность
- **Риск:** Сезонный коэффициент основан на квартальных данных; внутриквартальные колебания не учтены
- **Вероятность:** Низкая
- **Влияние:** Погрешность ±5-10% для сезонно-зависимых товаров
- **Митигация:** Ограничение коэффициента диапазоном [0.7, 1.5]; минимум 5 записей за квартал для расчёта

### 2.3 Малые выборки
- **Риск:** Для редких ENSTRU кодов выборка может быть <10 записей
- **Вероятность:** Высокая (для специфичных ТРУ)
- **Влияние:** Низкая статистическая значимость выводов
- **Митигация:** Уровень доверия: low при N<10; агент предупреждает пользователя

### 2.4 Ложные аномалии
- **Риск:** IQR и Isolation Forest могут давать false positives (нормальные закупки, отмеченные как аномальные)
- **Вероятность:** Средняя
- **Влияние:** Необоснованные подозрения
- **Митигация:** Consensus метод (пересечение IQR и IF) снижает false positives; severity levels дифференцируют серьёзность

## 3. Риски LLM

### 3.1 Галлюцинации
- **Риск:** LLM может генерировать факты, не подкреплённые данными из БД
- **Вероятность:** Низкая (при temperature=0.0 и strict tool use)
- **Влияние:** Неверные ответы, потеря доверия
- **Митигация:** temperature=0.0; system prompt запрещает домысливание; все вычисления делегируются SQL/Python

### 3.2 Точность извлечения фактов
- **Риск:** Точность может быть ниже целевых 85% для сложных составных вопросов
- **Вероятность:** Средняя
- **Влияние:** Несоответствие ТЗ
- **Митигация:** Evaluation pipeline с ground truth; итеративное улучшение system prompt; фиксированный формат ответа

### 3.3 Доступность LLM API
- **Риск:** nitec-ai.kz API может быть недоступен или отвечать медленно
- **Вероятность:** Низкая
- **Влияние:** Полная остановка агента
- **Митигация:** Таймаут 60с; retry с exponential backoff; возможность переключения на другой LLM провайдер через env vars

### 3.4 Многоязычность
- **Риск:** Качество ответов на казахском языке может быть ниже, чем на русском
- **Вероятность:** Средняя
- **Влияние:** Неполные или грамматически некорректные ответы на KZ
- **Митигация:** Основные данные хранятся в полях name_ru и name_kz; LLM определяет язык по вопросу

## 4. Инфраструктурные риски

### 4.1 Масштабирование БД
- **Риск:** PostgreSQL может замедлиться при росте данных >10M записей
- **Вероятность:** Низкая (текущий объём ~200K)
- **Влияние:** Медленные ответы агента
- **Митигация:** Materialized views; оптимальные индексы (20+); партиционирование (при необходимости)

### 4.2 Безопасность
- **Риск:** SQL injection через agent tool execute_sql
- **Вероятность:** Очень низкая
- **Влияние:** Утечка данных
- **Митигация:** Whitelist SELECT only; blacklist опасных операций; auto-LIMIT 100; read-only user (рекомендуется)

### 4.3 On-prem развёртывание
- **Риск:** Зависимость от внешнего LLM API (nitec-ai.kz) при on-prem деплое
- **Вероятность:** Средняя
- **Влияние:** Невозможность работы в изолированной сети
- **Митигация:** Конфигурация LLM через env vars; возможность использования локального LLM (Ollama, vLLM)
